## 一、项目现状

Impress 是一个双语（zh/en）React SPA + Go/Gin CMS。下表以当前仓库行为为准；后续章节保留原始需求池和产品推演，不再作为实现状态依据。

### 已有能力

| 模块 | 当前状态 | 能力 |
|------|----------|------|
| 统一页面 | production | `unified_pages` 统一承载草稿、版本、发布内容、公开 slug、导航与 SEO 元数据；支持发布、下线和回滚 |
| 文章内容 | production | 文章 CRUD、分类、标签、Markdown/Tiptap 编辑和公开读取 |
| 公开站点 | production | Bootstrap、统一页面动态路由、主题兼容补位、自动导航、Sitemap、RSS |
| 权限 | production | JWT + 细粒度 RBAC；主要后台 API 按 read/create/update/delete/publish/manage 收口 |
| 审计 | production | 登录和后台写操作写入数据库；发布、下线、回滚记录成功/失败、actor、resource 与请求元数据 |
| 事件/Webhook | production | 内容创建、更新、草稿更新、发布、下线、回滚、删除均有明确事件 |
| 媒体与备份 | production | 媒体上传/裁剪/目录管理；备份、导入导出接口与权限边界 |
| 管理端质量门禁 | production | lint、typecheck、前后端测试、Go race test、构建及 Playwright 核心发布链路 |
| 部署 | production | Docker + SSH/systemd，支持 SQLite / PostgreSQL |

### 受限或未完成能力

- `scheduledAt` 和旧调度器尚未接入统一页面 service，定时发布仍不属于发布承诺（Wave 1-C）。
- migration 与 system status 后端能力已存在，但正式产品入口、审计摘要和验收仍待 Wave 1-D。
- AI 向导、QA、翻译和远端存储配置仍存在运行时断链，保持 experimental/hidden。
- 多站点只有管理壳，核心内容尚未完成 `site_id` 数据隔离。
- Marketplace/Plugin 仍是骨架，尚未形成可安装、启停、升级、卸载的真实生命周期。
- 线索管理、预约咨询和商业化流程尚未形成完整产品闭环。

## 三、对标 Halo CMS：核心 vs 插件划分

### 3.1 Halo CMS 概况

[Halo](https://github.com/halo-dev/halo) 是 Java/Spring Boot 开源建站工具（38k Stars），采用"精简核心 + 可插拔扩展"哲学。

**Halo 核心内置：** 文章/页面 CRUD、分类/标签、附件管理、用户角色(RBAC)、主题引擎、默认编辑器、Lucene 全文搜索、备份恢复、REST API、Console 管理界面

**Halo 做成插件的：** Sitemap、RSS Feed、评论组件、代码高亮、OSS 存储、OAuth 登录、统计集成、编辑器替换、联系表单、Webhook、SEO 推送等（90+ 社区插件）

### 3.2 四大 CMS 核心/插件边界对比

| 维度 | WordPress | Ghost | Halo 2 | 印迹官网（建议） |
|------|-----------|-------|--------|------------------|
| SEO（sitemap/meta） | 插件 | **核心** | 插件 | **核心** |
| RSS Feed | **核心** | **核心** | 插件 | 插件 |
| 评论系统 | **核心** | **核心** | 插件 | 插件 |
| 搜索 | **核心** | **核心** | 核心+插件可替换 | 插件 |
| 统计分析 | 插件 | 插件 | 插件 | 插件 |
| 联系表单 | 插件 | 需集成 | 插件 | 插件 |
| 云存储 | 插件 | 需配置 | 插件 | 插件 |
| 编辑器 | 核心 | 核心 | 核心+插件可新增 | **核心** |
| 定时发布 | **核心** | **核心** | **核心** | **核心** |
| 备份恢复 | 插件 | 需配置 | **核心** | **核心** |

### 3.3 划分原则

Halo 能把 Sitemap、RSS 等做成插件是因为它有成熟的应用市场 + 社区生态（90+ 插件）。印迹官网是单一项目，没有插件市场，划分原则应不同：

> **印迹官网原则：宁可核心厚一点，也不要让常用功能变成"要自己写插件才能用"。**
>
> 更接近 **Ghost 哲学**（核心包含所有常用功能）而非 Halo（极致拆分）。

| 归属 | 判断标准 |
|------|---------|
| **核心功能** | CMS 不可或缺的基础能力；所有用户都需要；删除后系统不完整 |
| **插件** | 可选增强；仅部分用户/场景需要；与核心解耦可独立开关 |

## 六、插件架构设计

### 6.1 设计参考

借鉴 Halo 的扩展点机制，适配 Go + React 技术栈。核心概念：

- **MULTI_INSTANCE**：多个插件同时生效（如多个统计代码同时注入）
- **SINGLETON**：同时只有一个实现（如搜索引擎只选一个）

### 6.2 后端扩展点（Go/Gin）

```go
// 扩展点注册表
type ExtensionRegistry struct {
    // MULTI_INSTANCE: 多个同时生效
    headProcessors  []HeadProcessor
    eventListeners  []EventListener

    // SINGLETON: 同时只有一个
    searchEngine    SearchEngine
    storageProvider StorageProvider
}

// HeadProcessor — 向页面 <head> 注入 HTML 片段
// 用途：统计代码注入、Schema.org 结构化数据
type HeadProcessor interface {
    InjectHead(pageKey string) []string
}

// EventListener — 监听系统事件
// 用途：Webhook 通知、CDN 缓存刷新、邮件通知
type EventListener interface {
    OnEvent(event string, payload map[string]interface{})
}

// SearchEngine — 全文搜索引擎（SINGLETON）
// 用途：默认 SQLite FTS / 可替换为 Meilisearch
type SearchEngine interface {
    Index(doc SearchDocument) error
    Search(query string, limit int) ([]SearchResult, error)
}

// StorageProvider — 文件存储后端（SINGLETON）
// 用途：默认本地存储 / 可替换为 OSS/S3
type StorageProvider interface {
    Upload(file io.Reader, filename string) (url string, err error)
    Delete(url string) error
}
```

### 6.3 系统事件定义

```go
// 可被 EventListener 监听的事件
const (
    EventContentPublished = "content.published"   // 内容发布
    EventContentRollback  = "content.rollback"     // 内容回滚
    EventFormSubmitted    = "form.submitted"        // 表单提交
    EventMediaUploaded    = "media.uploaded"        // 媒体上传
    EventUserLogin        = "user.login"            // 用户登录
)
```

### 6.4 前端扩展点（React）

| 扩展点 | 类型 | 说明 | 对标 Halo |
|--------|------|------|-----------|
| `<HeadInjector />` | MULTI | 向 `<head>` 注入统计/SEO 标签 | TemplateHeadProcessor |
| `<AdminNavExtension />` | MULTI | 管理后台导航栏插件入口 | Plugin Detail Tabs |
| `<EditorToolbar />` | MULTI | 编辑器工具栏按钮扩展 | Default Editor Extension |
| `<PageFooterSlot />` | MULTI | 公共页面底部注入 | TemplateFooterProcessor |
| `<DashboardWidget />` | MULTI | 仪表盘自定义卡片 | Dashboard Widgets |

### 6.5 插件配置存储

```sql
-- 插件启用/禁用及配置通过 site_settings 表管理
CREATE TABLE site_settings (
    key   VARCHAR(100) PRIMARY KEY,
    value TEXT NOT NULL  -- JSON

-- 示例数据

```
